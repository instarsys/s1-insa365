## 1. Gusto 벤치마크 분석

### Gusto 핵심 UI 특징

| 항목 | Gusto 현황 | s1-insa365 적용 |
|------|-----------|----------------|
| **레이아웃** | 좌측 사이드바 + 중앙 콘텐츠 | 동일 구조 채택 |
| **메뉴 구조** | People / Pay / Time & Attendance / Benefits / Taxes & Compliance | 한국 환경 재구성 (섹션 5 참조) |
| **메뉴 원칙** | Flat hierarchy, 2클릭 이내 핵심 기능 도달 | 동일 원칙 |
| **대시보드** | To-do 리스트 중심, 최근 급여 상태 카드 | To-do + 위젯 (PRD 정의) |
| **급여 실행** | 스프레드시트 뷰 (DataGrid V2), Review Summary, Confirm | 3-Step (PRD 정의) |
| **디자인 시스템** | Workbench — Inter 폰트, 토큰 기반, 120+ 스크린, 37 컴포넌트 | 유사 규모 디자인 시스템 구축 |
| **컬러** | Green(#c8ed78) + Blue(#2469ff) + Purple(#6352fa), 밝은 톤 | Purple 메인으로 전환 |
| **폰트** | Inter Display / Inter (400~900) | Pretendard (한글) + Inter (영문/숫자) |
| **모바일 (직원)** | Gusto Wallet — 급여확인, 출퇴근, 예산 도구 | 하단 탭바 PWA (PRD 4.11 정의) |
| **일러스트** | 친근한 캐릭터 (돼지 로딩 등), 밝은 톤 일러스트 | 한국 감성 일러스트, Empty state에 활용 |
| **Rounded corners** | 16px~24px | 12px~16px (약간 절제) |

### Gusto에서 차용할 핵심 패턴

1. **To-do 대시보드**: 로그인 → 즉시 "할 일" 표시 (급여 마감, 미확정 근태, 미처리 휴가)
2. **스프레드시트 급여 입력**: 엑셀처럼 인라인 편집, 컬럼 커스텀, Import 지원
3. **Context switching 최소화**: 급여 중 직원 정보 → 슬라이드 패널 (새 페이지 이동 없음)
4. **2클릭 원칙**: 사이드바 → 서브메뉴로 핵심 액션 도달
5. **직원 셀프서비스**: 별도 모바일 최적화 UI (관리자 UI와 분리)
6. **Delight 요소**: 로딩 애니메이션, 빈 상태 일러스트, 완료 축하 모션

| Element | Font | Weight | Size | Line Height |
|---------|------|--------|------|-------------|
| Display | Pretendard | 700 (Bold) | 28px | 36px |
| H1 (페이지 제목) | Pretendard | 600 (SemiBold) | 24px | 32px |
| H2 (섹션 제목) | Pretendard | 600 | 20px | 28px |
| H3 (카드 제목) | Pretendard | 600 | 16px | 24px |
| Body (본문) | Pretendard | 400 | 14px | 20px |
| Body Small | Pretendard | 400 | 13px | 18px |
| Caption | Pretendard | 400 | 12px | 16px |
| Label | Pretendard | 500 (Medium) | 12px | 16px |
| Button | Pretendard | 500 | 14px | 20px |
| Numbers (금액) | Inter | 500 | 14px | 20px |
| Spreadsheet Cell | Inter | 400 | 13px | 18px |

**폰트 선택 근거**:
- **Pretendard**: 한글 최적화, Apple SD Gothic Neo 계열. 한글 본문에서 뛰어난 가독성.
- **Inter**: 숫자/금액 표시 시 tabular figures 지원. 급여 스프레드시트에서 숫자 정렬 최적화.

**적용 규칙**:
- 모든 한글 텍스트: Pretendard
- 금액, 통계 숫자, 스프레드시트 셀: Inter
- 영문 혼용 시: Pretendard (한글 우선, Inter fallback)

## 5. 관리자(Admin) 사이드바 메뉴 구조

### Gusto vs s1-insa365 메뉴 매핑

| Gusto 메뉴 | s1-insa365 메뉴 | 차이점 |
|-----------|----------------|--------|
| Home (Dashboard) | 대시보드 | To-do 리스트 중심 (동일) |
| People | 직원 관리 | 직원 목록, 부서/직급 관리 |
| Pay → Run Payroll | 급여 관리 → 급여 실행 | 3-Step 프로세스 (한국 로컬라이즈) |
| Pay → Payment History | 급여 관리 → 급여 이력 | 동일 개념 |
| Time & Attendance | 근태 관리 | 한국 52시간제 추가 |
| Benefits | *Phase 2+* | 한국은 4대보험 자동 (별도 메뉴 불필요) |
| Taxes & Compliance | 세무/신고 | 한국 세무 (원천징수, 간이지급명세서) |
| *없음* | 리포트 | 한국 급여대장/부서별 인건비 등 별도 필요 |
| Settings | 설정 | 동일 |

### 최종 메뉴 트리 (PRD 4.9 기반)

```
대시보드                    → /dashboard
직원 관리                   → /employees
  ├─ 직원 목록              → /employees/list
  ├─ 부서/직급 관리          → /employees/departments
  └─ 일괄 업로드            → /employees/import
근태 관리                   → /attendance
  ├─ 일별 근태              → /attendance/daily
  ├─ 월별 현황              → /attendance/monthly
  ├─ 휴가 관리              → /attendance/leave
  └─ 52시간 모니터링         → /attendance/overtime
급여 관리                   → /payroll
  ├─ 급여 실행              → /payroll/run (3-Step)
  ├─ 급여 이력              → /payroll/history
  ├─ 급여대장               → /payroll/ledger
  └─ 급여명세서             → /payroll/payslips
세무/신고                   → /tax
  ├─ 원천징수 현황           → /tax/withholding
  └─ 신고 데이터            → /tax/reports
리포트                      → /reports
  ├─ 급여 리포트            → /reports/payroll
  └─ 근태 리포트            → /reports/attendance
설정                        → /settings
  ├─ 회사 정보              → /settings/company
  ├─ 근무 정책              → /settings/work-policy
  ├─ 급여 규칙              → /settings/salary-rules
  └─ 플랜/결제              → /settings/billing
시스템 관리 (SYSTEM_ADMIN)   → /system
  ├─ 법정 파라미터           → /system/legal-params
  └─ 감사 로그              → /system/audit-log
```

| Breakpoint | Sidebar | 콘텐츠 | 비고 |
