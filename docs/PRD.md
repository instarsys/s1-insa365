## 1. 제품 개요

### 1.1 비전

> "한국 중소기업의 급여를 3분 안에"

한국 노동법의 복잡한 급여 계산(4대보험, 간이세액표, 연장/야간/휴일 가산)을 자동화하여, 중소기업 HR 담당자의 월말 급여 업무를 반나절에서 3분으로 단축한다.

### 1.2 미션

- Excel 기반 급여 관리를 클라우드 SaaS로 전환
- 4대보험 요율/간이세액표 변경 시 자동 반영
- 근태 데이터와 급여 계산의 완전 자동 연동
- 52시간제 컴플라이언스 기본 내장

### 1.3 타겟 시장

| 구분 | 내용 |
|------|------|
| 시장 | 한국 중소기업 HR SaaS |
| 기업 규모 | 50~300명 |
| 타겟 산업 | 전 업종 (제조, IT, 서비스, 유통) |
| 지역 | 대한민국 (한국 노동법 특화) |
| TAM | 한국 중소기업 ~70만개 × 평균 20만원/월 = ~1.7조원/년 |
| SAM | 50~300명 규모 ~8만개 × 20만원/월 = ~1,920억원/년 |
| SOM (3년) | 500개사 × 25만원/월 = ~15억원/년 |

### 1.4 경쟁 분석

#### 직접 경쟁자

| 제품 | 포지셔닝 | 강점 | 약점 |
|------|---------|------|------|
| **Flex** | 종합 HR 플랫폼 | 기능 풍부, 브랜드 파워 | 높은 가격, 복잡한 도입 |
| **Shopl** | 근태 특화 | 모바일 UX, 현장직 | 급여 기능 미약 |
| **Shiftee** | 시프트/스케줄 | 편의점/유통 특화 | SME 급여에 부적합 |
| **더존 iCUBE** | 전통 ERP | 세무사 연동, 실적 | 구식 UX, 높은 비용 |
| **더존 WEHAGO** | 클라우드 ERP/HR | 세무사 80%+ 사용, 생태계(NAHAGO) | 구식 UX, 학습 곡선, 50명 기준 ~32만원/월 |
| **Newploy** | 급여 자동화 | RPA 급여이체, 급여 특화 | 규모 작음, 기능 제한 |
| **Smart A** | 레거시 급여 | 기존 고객 기반 | **2025.12 단종 예정** (이전 기회) |

#### 간접 경쟁자

| 제품 | 위협 |
|------|------|
| **Excel/구글 시트** | 무료, 관성. 하지만 오류 발생률 높음 |
| **세무사 대행** | 신뢰도. 하지만 비용 높고 실시간 불가 |
| **자체 개발** | 커스터마이징. 하지만 유지보수 부담 |

#### 차별화 전략

```
s1-insa365 = Gusto의 UX 철학 + 한국 노동법 전문성
```

1. **급여 특화**: 기능 범위를 좁히고 급여 정확도/UX에 올인
2. **셀프 온보딩**: 영업팀 미팅 없이 3분 내 시작
3. **투명 가격**: 기본료 + 인당 요금, 숨은 비용 없음
4. **자동 컴플라이언스**: 4대보험 요율, 세율 변경 시 자동 반영
5. **Smart A 이탈 공략**: 2025.12 유지보수 종료 → 이전 대상 기업 타겟 마케팅
6. **세무사 파트너 전략**: 세무사 도구 무료 제공 → 수임회사 유입 채널 (Gusto 회계사 파트너 모델)
7. **정부 바우처 활용**: 비대면 서비스 바우처 70% 보조금으로 도입 장벽 제거

## 2. 사용자 페르소나

### 페르소나 1: HR 담당자 김유진 (Primary User)

| 항목 | 내용 |
|------|------|
| 역할 | 150명 제조업체 인사/급여 담당 |
| 나이 | 32세 |
| IT 숙련도 | 중간 (Excel 능숙, SaaS 경험 적음) |
| 현재 도구 | Excel + 세무사 |
| 주요 Pain | 월말 급여 계산에 2일 소요, 4대보험 요율 변경 누락 불안, 직원 문의 대응 |
| 원하는 것 | 자동 계산으로 실수 제거, 직원에게 급여 상세 투명 제공 |
| 두려운 것 | 새 시스템 학습 부담, 기존 데이터 마이그레이션 |

**핵심 니즈**: "급여 계산을 정확하게, 빨리 끝내고 싶다"

| 두려운 것 | 급여 오류로 인한 노무 분쟁, 직원 불신 |

**핵심 니즈**: "비용은 줄이면서 컴플라이언스는 확보하고 싶다"

## 4. 기능 상세 스펙

### 4.1 직원 관리 모듈

#### 4.1.1 직원 등록

**화면**: 직원 등록 모달/페이지

**입력 필드**:

| 필드 | 타입 | 필수 | 유효성 검증 | 비고 |
|------|------|------|------------|------|
| 이름 | text | O | 2~20자, 한글/영문 | |
| 이메일 | email | O | 이메일 형식, 중복 체크 | 로그인 ID |
| 연락처 | tel | O | 한국 전화번호 형식 | |
| 주민등록번호 | text | X | 13자리, 형식 검증 | AES-256 암호화 저장 |
| 부서 | select | O | 회사 부서 목록 | |
| 직급 | select | O | 회사 직급 목록 | |
| 입사일 | date | O | 미래 날짜 가능 | |
| 급여 구분 | select | O | 고정급/시급제/일급제 | |
| 근무 정책 | select | O | 회사 근무정책 목록 | |
| 근무지 | select | O | 회사 근무지 목록 | |

**사번 자동 생성**: `E{영문자1}{숫자4}` (예: EA1234)
- 영문자: A-Z 랜덤
- 숫자: 0000-9999 랜덤
- 중복 체크 후 재생성

**급여 항목 설정**:
- 직원 등록 시 회사 SalaryRule 템플릿에서 EmployeeSalaryItem으로 복사
- 기본급(BASE_SALARY), 시급(BASE_HOURLY), 일급(BASE_DAILY) 필수 입력
- 기본시급 자동계산: `기본급 / 209` (209 = 월 소정근로시간)
- **통상시급**: 기본시급과 구별. 통상임금 해당 수당(isOrdinaryWage=true)을 합산하여 별도 산정
- 각 수당 항목에 `isOrdinaryWage` 플래그 설정 (2024 대법원 판결 반영)
- 각 수당 항목에 `paymentCycle` 설정 (격월/분기/연간 수당의 월환산 기준)
- 비과세 수당은 `taxExemptCode`와 `taxExemptLimit` 설정

#### 4.1.2 직원 목록/검색

**화면**: 직원 관리 메인 페이지

| 기능 | 상세 |
|------|------|
| 목록 표시 | 사번, 이름, 부서, 직급, 입사일, 상태 |
| 검색 | 이름, 사번, 이메일 통합 검색 |
| 필터 | 부서, 직급, 재직 상태 (재직/퇴사/전체) |
| 정렬 | 이름, 입사일, 부서 (오름/내림) |
| 페이지네이션 | 20/50/100건 선택 |
| 일괄 작업 | 선택 직원 부서 변경, 근무정책 변경 |

#### 4.1.3 직원 상세/수정

**화면**: 직원 상세 페이지 (탭 구성)

| 탭 | 내용 |
|----|------|
| 기본정보 | 인적사항, 연락처, 부서/직급 |
| 급여정보 | 기본급, 수당/공제 항목, 보험 모드 |
| 근태이력 | 최근 3개월 근태 요약 |
| 급여이력 | 최근 6개월 급여 내역 |

#### 4.1.4 입퇴사 관리

| 처리 | 동작 |
|------|------|

### 4.3 급여 계산 모듈

#### 4.3.1 급여 계산 엔진

**계산 순서 (6단계)**:

```
Phase 0: 기초 데이터 로딩
   ├── EmployeeSalaryItem → 기본급, 수당/공제 항목
   ├── SalaryAttendanceData → 근태 스냅샷
   └── InsuranceRate, TaxBracket, TaxExemptLimit → 요율/세율/비과세 한도
   └── Company 설정 (prorationMethod, 야간시간 등)

Phase 1: 통상임금 산정 ★ (2024.12.19 대법원 전원합의체 판결 반영)
   ├── 기본급 + isOrdinaryWage=true인 수당 합산
   ├── 격월/분기/연간 수당 월환산 (격월÷2, 분기÷3, 반기÷6, 연간÷12)
   ├── 통상시급 = 월 통상임금 / 209
   └── 최저임금 검증 (통상시급 < 최저시급 시 경고)

Phase 2: 총 지급액 계산
   ├── 기본급 (중도입퇴사 시 일할계산 — prorationMethod에 따라)
   ├── + 고정 수당 (식대, 직책수당 등)
   ├── + 계산식 수당 (통상시급 기반 가산수당)
   │   ├── 연장: 통상시급 × 1.5 × 연장시간/60
   │   ├── 야간: 통상시급 × 0.5 × 야간시간/60
   │   ├── 야간+연장: 통상시급 × 2.0 × 중복시간/60
   │   ├── 휴일(8h이내): 통상시급 × 1.5 × 휴일근로시간_8h이내/60
   │   ├── 휴일(8h초과): 통상시급 × 2.0 × 휴일근로시간_8h초과/60
   │   ├── 휴일+야간(8h이내): 통상시급 × 2.0 × 시간/60
   │   └── 휴일+야간(8h초과): 통상시급 × 2.5 × 시간/60
   ├── + 변동 수당 (성과금 등)
   ├── + 주휴수당 (시급/일급제: 주 15h 이상 시)
   ├── - 근태 공제 (지각, 조퇴, 결근, 외출)
   └── = totalPay

Phase 3: 비과세 분리
   ├── 항목별 비과세 한도 적용 (TaxExemptLimit 조회)
   │   ├── 식대: 월 200,000원 한도
   │   ├── 차량유지비: 월 200,000원 한도
   │   └── 보육수당: 월 200,000원 한도 (초과분 → 과세)
   ├── 과세 대상 소득 = totalPay - 비과세 합계
   └── 4대보험 기준보수 산정

Phase 4: 공제액 계산
   ├── 국민연금: clamp(기준소득, 하한, 상한) × 4.5% (10원 미만 절사)
   ├── 건강보험: 보수월액 × 3.545% (원 미만 절사)
   ├── 장기요양: 건강보험료 × 12.95% (원 미만 절사)
   ├── 고용보험: 보수 × 0.9% (원 미만 절사)
   ├── 소득세: TaxBracket 조회 (과세소득 + 부양가족수)
   ├── 지방소득세: 소득세 × 10%
   ├── + 기타 공제 (조합비, 대출상환 등)
   └── = totalDeduction

Phase 5: 실수령액
   └── netPay = totalPay - totalDeduction
```

**통상임금 정의 (2024 대법원 판결)**:
- 소정근로의 대가로 정기적·일률적으로 지급하는 임금 (고정성 요건 폐기)
- 재직조건부 상여금, 근무일수 조건부 수당, 성과급 최소보장분 → 통상임금 포함
- 통상시급 = 월 통상임금 / 209시간

**중도입퇴사 일할계산**:
| 방식 | 계산 | 비고 |
|------|------|------|
| 역일 기준 (CALENDAR_DAY) | (월급 / 해당월 역일수) × 재직일수 | 기본값 |
| 소정근로일 기준 (WORKING_DAY) | (월급 / 해당월 소정근로일수) × 근무일수 | 회사 선택 |

**최저임금 검증**:
| 연도 | 시급 | 월급 환산 (209h) | 비고 |
|------|------|----------------|------|
| 2025 | 10,030원 | 2,096,270원 | |
| 2026 | 10,320원 | 2,156,880원 | |
- 수습기간: 1년 이상 계약 + 단순노무직 제외 시, 3개월간 90% 적용

```
결근일수, 지각시간, 조퇴시간, 외출시간,

```

**가산수당 종합표** (근로기준법 제56조):

| 근로 유형 | 8시간 이내 | 8시간 초과 | 비고 |
|----------|-----------|-----------|------|
| 연장근로 | 1.5배 | 1.5배 | |
| 야간근로 (22:00~06:00, 정규 내) | +0.5배 | +0.5배 | 정규시간 중 야간 |
| 연장+야간 | 2.0배 | 2.0배 | 중복가산 |

| 휴일+야간 | 2.0배 | 2.5배 | 중복가산 |
| 휴일+연장+야간 | - | 2.5배 | |

**계산식 예시** (통상시급 기반):
| 항목 | 수식 |
|------|------|
| 연장근로수당 | `F((통상시급)*1.5*(연장근로시간)/60)` |
| 야간근로수당 | `F((통상시급)*0.5*(야간근로시간)/60)` |
| 야간+연장수당 | `F((통상시급)*2.0*(야간연장중복시간)/60)` |
| 휴일근로수당(8h이내) | `F((통상시급)*1.5*(휴일근로시간_8h이내)/60)` |
| 휴일근로수당(8h초과) | `F((통상시급)*2.0*(휴일근로시간_8h초과)/60)` |
| 휴일+야간수당(8h이내) | `F((통상시급)*2.0*(휴일야간시간_8h이내)/60)` |
| 휴일+야간수당(8h초과) | `F((통상시급)*2.5*(휴일야간시간_8h초과)/60)` |
| 주휴수당 (시급제) | `F((통상시급)*8*(주휴일수))` |
| 주휴수당 (단시간) | `F((통상시급)*(실주근로시간/40)*8*(주휴일수))` |
| 지각/조퇴공제 | `F((통상시급)*(지각시간+조퇴시간)/60)` |
| 결근공제 | `F((일급)*(결근일수))` |

### E. 퇴직금 계산 설계 (Phase 1.5)

**기본 공식**:
```
퇴직금 = 1일 평균임금 × 30일 × (총 재직일수 / 365)

1일 평균임금 = 퇴직일 이전 3개월간 임금 총액 / 해당 3개월간 역일수

3개월간 임금 총액 = (A) 3개월간 기본 급여 합계
                   + (B) 퇴직 전 12개월간 상여금 총액 × 3/12
                   + (C) 퇴직 전 12개월간 연차수당 × 3/12

if 평균임금 < 통상임금: 적용임금 = 통상임금 (근로기준법 제2조 제2항)
```

**평균임금 포함/제외**:

| 포함 | 제외 |
|------|------|
| 기본급, 정기수당 (직무/근속/직책) | 경조사비, 출장실비 |
| 식대/교통비 (과세분) | 학자금 지원 |
| 정기상여금 (12개월→3/12 환산) | 성과급 (비정기) |
| 연차미사용수당 (12개월→3/12) | 차량유지비 (실비변상) |
| 연장/야간/휴일수당 | 생일축하금 등 은혜적 급여 |

**퇴직소득세 계산 (8단계)**:
```
① 퇴직소득금액 = 퇴직급여액 - 비과세 퇴직소득
② 근속연수공제 (5년 이하: 연 100만, 5~10년: +200만, 10~20년: +250만, 20년+: +300만)
③ 환산급여 = (퇴직소득금액 - 근속연수공제) / 근속연수 × 12
④ 환산급여공제 (800만 이하 100%, 이후 60%→55%→45%→35%)
⑤ 환산과세표준 = 환산급여 - 환산급여공제
⑥ 환산산출세액 = 환산과세표준 × 누진세율 - 누진공제
⑦ 산출세액 = 환산산출세액 / 12 × 근속연수
⑧ 지방소득세 = 산출세액 × 10%
```

**법적 요구사항**:
- 지급 기한: 퇴사일로부터 14일 이내 (지연 시 연 20% 이자)
- IRP 의무이전: 원칙적으로 IRP 계좌로 (예외: 55세 이후, 300만원 이하, 사망, 외국인 출국)
- 대상: 1년 이상 근속 직원 (파트타임은 주 15시간 이상)
- 퇴직소득 지급명세서: 다음해 3월 10일까지 국세청 제출

**필수 데이터**: 급여 지급 이력 (최소 12개월), 상여금 이력 (12개월), 연차수당 이력, 중간정산 이력

### F. 세무사 파트너 전략 (Phase 1.5~2)

**한국 세무사 시장 현황**:
- 전국 등록 세무사: ~17,000명 (2025)
- 세무법인: 500~700개, 총매출 3.7조원
- 세무사 사용 소프트웨어: 더존 WEHAGO T (80%+), 세무사랑Pro (72%)
- 급여 아웃소싱 수수료: 인당 월 7,000~17,000원

**세무사 Pain Points**:

| Pain Point | 심각도 |
|-----------|--------|
| 매월 전화/FAX/카카오로 수작업 데이터 수집 | 최고 |
| 엑셀 데이터를 세무프로그램에 재입력 | 매우 높음 |
| 법령 변경(요율/세율) 지속 업데이트 부담 | 높음 |
| 다중 시스템 사용 (WEHAGO+홈택스+EDI+은행) | 중간 |

**파트너 프로그램 설계**:

| 티어 | 요건 | 수임처 할인 | 수익 공유 |
|------|------|-----------|----------|
| 스타터 | 가입만 | - | - |
| 실버 | 수임처 5개+ | 10% | 5% |
| 골드 | 수임처 15개+ | 15% | 10% |
| 플래티넘 | 수임처 30개+ | 20% | 15% |

**세무사에게 제공할 핵심 가치**:
1. **수임회사 데이터 자동 수집** ← Killer Feature (전화/FAX 제거)
2. **원천세 신고 자동화** (WEHAGO 호환 Export + 홈택스 파일 생성)
3. **세무사 대시보드** (전체 수임회사 급여 현황 한눈에)
4. **무료 사용** (세무사 자체 사무실 급여)
5. **세무사 디렉토리** (고객 소개)

**GTM 단계**:
- Phase 1.5 (0~6개월): 무료 대시보드, WEHAGO 호환, 원천세 자동화
- Phase 2 (6~12개월): 파트너 프로그램, 세무사회 협력, 성공 사례
- Phase 3 (12~24개월): 생태계 — 세무사 디렉토리, 4대보험 EDI 통합

### G. 연말정산 설계 방향 (Phase 2)

**연말정산 타임라인**:

| 시기 | 업무 |
|------|------|
| 11~12월 | 간소화 일괄제공 명단 등록, 직원 동의 수집 |
| 1월 15일 | 국세청 간소화서비스 개통 (45개 항목) |
| 1월 20일~ | 최종 확정 자료, 일괄 PDF 다운로드 가능 |
| 1월~2월 | 직원 추가 서류 제출 → 공제 계산 → 환급/추가징수 |
| 2월 급여 | 환급/추가납부 반영 |
| 3월 10일 | 근로소득 지급명세서 국세청 제출 마감 |

**연말정산 계산 (11단계 요약)**:
```
① 총급여액 = 연간 근로소득 - 비과세소득 (PayrollMonthly 12개월 합산)
② 근로소득공제 (70%~2%, 한도 2,000만원)
③ 근로소득금액 = 총급여액 - 근로소득공제
④ 종합소득공제 (인적, 연금보험료, 특별소득, 기타소득)
⑤ 과세표준
⑥ 산출세액 (6%~45%, 8개 구간)
⑦ 세액감면 (중소기업 취업자 등)
⑧ 세액공제 (근로소득, 자녀, 연금, 보험료, 의료비, 교육비, 기부금, 월세 등)
⑨ 결정세액 = 산출세액 - 감면 - 공제
⑩ 기납부세액 = PayrollMonthly 연간 원천징수 합계
⑪ 차감징수세액 = 결정세액 - 기납부세액 (양수: 추가납부, 음수: 환급)
```

**MVP에서 준비해야 할 것**:
- PayrollMonthly 테이블에 월별 과세/비과세/원천징수 이력 저장 (이미 반영)
- EmployeeSalaryItem에 비과세 코드(taxExemptCode) 관리 (이미 반영)
- 이 2가지만 있으면 세무사가 연말정산을 대행할 수 있음

**Phase 2 데이터 모델** (추후 상세화):
- `YearEndSettlement`: 연말정산 마스터 (상태: COLLECTING → CALCULATING → CONFIRMED)
- `YtaDependents`: 부양가족/인적공제
- `YtaIncomeDeductions`: 소득공제 항목별
- `YtaTaxCredits`: 세액공제 항목별
- `YtaSimplifiedData`: 간소화자료 원본/파싱 결과
- `YtaResultSummary`: 원천징수영수증
