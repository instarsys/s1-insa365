# s1-insa365 MVP 정의서

> **"한국 중소기업의 급여를 3분 안에"**
> Gusto의 "Payroll in minutes" 전략을 한국 노동법 환경에 맞게 재설계

---

## 1. 비전 & 미션

### 비전
한국 중소기업이 급여와 인사를 가장 쉽고 정확하게 처리할 수 있는 플랫폼

### 미션
- **Excel 급여 관리를 SaaS로 전환**: 수작업 오류를 제거하고, 법 개정에 자동 대응
- **노동법 컴플라이언스 자동화**: 4대보험, 간이세액표, 52시간제를 시스템이 알아서 처리
- **3분 급여 확정**: 월말 급여 업무를 반나절에서 3분으로 단축

### Gusto에서 차용하는 핵심 원칙
| Gusto 원칙 | 한국 적용 |
|-----------|---------|
| "Payroll in 3 clicks" | 급여 확정까지 3단계 (근태확인 → 검토 → 확정) |
| 투명한 가격 | 기본료 + 인당 요금, 숨은 비용 없음 |
| 셀프 온보딩 | 가입 후 3분 내 첫 급여 시뮬레이션 가능 |
| Delight in details | 한국어 UX, 한국 달력, 공휴일 자동 반영 |
| Compliance built-in | 4대보험 요율/세율 DB 내장, 매년 자동 업데이트 |

## 2. 타겟 고객

### Primary: 50~300명 규모 한국 중소기업

**현재 상태 (Pain Points)**:
- Excel로 급여 계산 → 수식 오류, 버전 관리 불가
- 4대보험 요율 변경 시 수동 업데이트 → 과납/미납 발생
- 근태 기록이 종이/엑셀 → 연장수당 계산 분쟁
- 52시간제 관리 부재 → 노동부 감사 리스크

**원하는 상태**:
- 월말에 "확인" 한 번으로 급여 확정
- 법 개정 시 시스템이 자동 반영
- 근태 → 급여 자동 연동으로 분쟁 제로
- 52시간 초과 사전 경고

### 타겟이 아닌 고객 (MVP 단계)
- 10명 미만 소기업 (비용 대비 효과 낮음)
- 300명 초과 대기업 (SAP/Oracle ERP 사용)
- 해외 법인 (한국 노동법 특화)

## 3. MVP 핵심 기능

### 원칙: "급여 정확도 99%+" 달성에 필요한 것만

```
┌─────────────────────────────────────────────┐
│              MVP 범위 (3개 모듈)              │
│                                             │
│  ┌─────────┐  ┌─────────┐  ┌─────────────┐ │
│  │ 직원관리 │→│ 근태관리 │→│ 급여 자동계산 │ │
│  └─────────┘  └─────────┘  └─────────────┘ │
│                                             │
│  + 인증/권한 + 대시보드 + 회사 설정           │
└─────────────────────────────────────────────┘
```

### 3-1. 급여 자동계산 (핵심 가치)

> Gusto가 "급여를 쉽게"로 시작했듯, 급여 계산 정확도가 MVP의 성패를 결정

**필수 기능**:
| 기능 | 설명 | 우선순위 |
|------|------|---------|
| 4대보험 자동계산 | 국민연금, 건강보험, 장기요양, 고용보험 요율 DB 내장 | P0 |
| 간이세액표 | 부양가족 수 기반 소득세/지방소득세 자동 조회 | P0 |
| 연장/야간/휴일수당 | 근태 데이터 기반 자동계산 (1.5배/0.5배 가산) | P0 |
| 수당/공제 규칙 엔진 | 회사별 커스텀 수당/공제 항목 설정 (계산식 지원) | P0 |
| 급여 명세 상세 | 지급/공제 항목별 내역 표시 | P0 |
| 일괄 급여 확정 | 전 직원 급여 한 번에 검토/확정 | P0 |
| 보험 모드 선택 | 자동/수동/미적용 (직원별 4대보험 선택) | P1 |
| 통상임금 산정 | 기본급 + 통상임금 해당 수당 합산, 통상시급 계산 (2024 대법원 판결 반영) | P0 |
| 중도입퇴사 일할계산 | 역일/소정근로일 기준 선택, 테넌트별 설정 | P0 |
| 최저임금 검증 | 통상시급 vs 최저임금 비교, 미달 시 경고 | P0 |
| 주휴수당 (시급/일급제) | 주 15시간 이상 근무 시 주휴수당 자동계산 | P0 |
| 비과세 분리 | 항목별 비과세 한도 적용, 과세/비과세 소득 분리 | P0 |
| 3-Step 급여 실행 (스프레드시트 UI) | Step 1: 스프레드시트 급여 데이터 확인/입력, Step 2: 검토 요약 (Review Summary), Step 3: 급여 확정 | P0 |
| 직원 건너뛰기 (Skip) | 휴직/무급휴가 직원 당월 급여 건너뛰기, 사유 기록 | P0 |
| 급여 이력 페이지 | 월별 전사 급여이력, 부서별 집계, Excel 내보내기, 전월 대비 변동 | P0 |
| 급여대장 | 급여 확정 시 자동 생성, 전 직원 상세/집계, Excel 내보내기 | P0 |

**계산 흐름**:
```
Phase 0: 기초 데이터 로딩
  ├── EmployeeSalaryItem → 기본급, 수당/공제 항목
  ├── SalaryAttendanceData → 근태 스냅샷
  └── InsuranceRate, TaxBracket, TaxExemptLimit → 요율/세율/비과세 한도

Phase 1: 통상임금 산정 ★ (2024 대법원 판결 반영)
  ├── 기본급 + isOrdinaryWage=true인 수당 합산
  ├── 격월/분기/연간 수당 월환산
  ├── 통상시급 = 월 통상임금 / 209
  └── 최저임금 검증 (경고)

Phase 2: 총 지급액 계산
  ├── 기본급 (중도입퇴사 시 일할계산)
  ├── + 고정 수당 (식대, 직책수당 등)
  ├── + 계산식 수당 (통상시급 기반)
  │   ├── 연장: 통상시급 × 1.5 × 연장시간
  │   ├── 야간: 통상시급 × 0.5 × 야간시간
  │   ├── 휴일(8h이내): 통상시급 × 1.5
  │   ├── 휴일(8h초과): 통상시급 × 2.0
  │   └── 중복가산: 야간+연장(2.0배), 휴일+야간(2.0/2.5배)
  ├── + 변동 수당, 주휴수당(시급/일급제)
  ├── - 근태 공제 (지각, 조퇴, 결근, 외출)
  └── = totalPay

Phase 3: 비과세 분리
  ├── 항목별 비과세 한도 적용 (초과분 → 과세)
  ├── 과세 대상 소득 = totalPay - 비과세 합계
  └── 4대보험 기준보수 산정

Phase 4: 공제액 계산
  ├── 국민연금: clamp(기준소득, 하한, 상한) × 4.5% (10원 미만 절사)
  ├── 건강보험: 보수월액 × 3.545% (원 미만 절사)
  ├── 장기요양: 건강보험료 × 12.95% (원 미만 절사)
  ├── 고용보험: 보수 × 0.9% (원 미만 절사)
  ├── 소득세: TaxBracket 조회
  ├── 지방소득세: 소득세 × 10%
  └── = totalDeduction

Phase 5: 실수령액
  └── netPay = totalPay - totalDeduction
```

**s0에서 배운 교훈 반영**:
- 세율/보험료율은 반드시 DB에서 관리 (프론트 하드코딩 금지)
- 급여 데이터는 단일 소스 (EmployeeSalaryItem) 에서만 관리
- 보험 모드(AUTO/MANUAL/NONE) 3가지 지원으로 예외 처리

### 3-2. 근태 관리

> 52시간제 컴플라이언스 + 급여 연동이 핵심

**필수 기능**:
| 기능 | 설명 | 우선순위 |
|------|------|---------|
| GPS 출퇴근 기록 | 직원 앱에서 위치 확인 후 출퇴근 체크 | P0 |
| 근무시간 자동계산 | 정규/연장/야간/휴일 자동 분류 | P0 |
| 52시간 추적 | 주 52시간 초과 실시간 모니터링 | P0 |
| 관리자 일괄 확정 | 월별 근태 데이터 일괄 검토/확정 | P0 |
| 근무 정책 설정 | 출퇴근 시간, 휴게시간, 유예시간 설정 | P0 |
| 급여 데이터 생성 | 확정된 근태 → 급여 계산용 스냅샷 생성 | P0 |
| 수동 입력/수정 | 관리자가 직접 근태 입력/수정 | P1 |
| 기본 휴가 관리 | 휴가 신청/승인/거절, 유급/무급 구분, 잔여연차 수동설정 | P0 |

**근태 → 급여 연동 플로우**:
```
출퇴근 기록 (매일)
  ↓
근태 확인/수정 (관리자)
  ↓
일괄 확정 (월말)
  ↓
급여 데이터 생성 (스냅샷)
  ↓
급여 자동계산 연동
```

**s0에서 배운 교훈 반영**:
- 근태 → 급여 데이터는 버전 관리 (재생성 가능)
- 외출 기록도 급여 공제에 반영 (processOutingRecords 설정)
- 유예시간(lateGraceMinutes) 반드시 지원

### 3-3. 직원 관리

> 직원 DB가 근태/급여의 기반 데이터

**필수 기능**:
| 기능 | 설명 | 우선순위 |
|------|------|---------|
| 직원 등록/수정 | 기본정보, 부서, 직급, 입사일 | P0 |
| 급여 항목 설정 | 직원별 기본급, 수당, 공제 항목 설정 | P0 |
| 부서/직급 관리 | 조직 구조 설정 | P0 |
| 직원 목록/검색 | 전체 직원 조회, 필터, 검색 | P0 |
| 입퇴사 관리 | 입사/퇴사 처리, 상태 관리 | P0 |
| 주민등록번호 암호화 | AES-256-GCM으로 PII 보호 | P0 |
| 직원 일괄 업로드 | Excel 파일로 대량 등록 | P1 |

**s0에서 배운 교훈 반영**:
- 사번 형식 `E{영문자1}{숫자4}` (충돌 방지)
- Soft delete로 퇴사 처리 (`deletedAt`)
- 급여 데이터는 User 모델이 아닌 별도 테이블(EmployeeSalaryItem)에서 관리

### 3-4. 공통 기반 (MVP 필수)

| 기능 | 설명 |
|------|------|
| 인증/권한 | JWT 기반, 역할별 접근제어 (SYSTEM_ADMIN, COMPANY_ADMIN, MANAGER, EMPLOYEE) |
| 다중 테넌시 | companyId 기반 데이터 격리 |
| 대시보드 (To-do 중심) | To-do 리스트(급여 마감, 미확정 근태, 미처리 휴가 신청, 52시간 초과 등) + 급여 요약, 근태 현황, 52시간 경고 위젯 |
| 회사 설정 | 근무 정책, 급여 규칙, 보험 설정 |
| 셀프 온보딩 | 가입 → 기본 설정 → 직원 등록 → 첫 급여 시뮬레이션 (3분) |
| 회사 가입 시드 데이터 | 기본 부서(5), 직급(5), 근무정책(1), 수당(11)/공제(12) 자동 생성 |
| 급여명세서 법적 필수항목 | 근로기준법 제48조 필수 기재사항 준수, 웹 조회 제공 |
| 원천징수이행상황신고서 데이터 | 월별 소득구분별 인원/지급액/세액 데이터 생성 (JSON) |
| 월별 과세/비과세/원천징수 이력 | PayrollMonthly 테이블에 연말정산 대비 월별 이력 저장 |
| SYSTEM_ADMIN 법정 파라미터 관리 | 4대보험 요율, 간이세액표, 비과세 한도, 최저임금, 법정 상수 CRUD + Excel 업로드(세액표) + 감사 로그 |
| 변경 감사 로그 (AuditLog) | 법정 파라미터 변경 이력 자동 기록, 변경 시 COMPANY_ADMIN 인앱 알림 |
| 기본 리포트 (4종) | 급여 요약, 부서별 인건비, 4대보험 현황, 근태 요약 리포트 (Excel/CSV 내보내기) |
| 글로벌 사이드바 메뉴 | Flat hierarchy 2단계 사이드바, 역할별 메뉴 노출, Action-centric 메뉴명 |
| 직원 모바일 화면 (PWA) | 하단 탭바(홈/근태/휴가/급여/MY), 홈에서 출퇴근+알림+급여 요약 |
| 직원 프로필 (MY) | 내 정보 확인, 연락처 수정, 비밀번호 변경, 4대보험 정보 확인 |
| 인앱 알림 (직원) | 급여 확정, 휴가 승인/거절, 52시간 주의, 근태 이상 알림 |

## 6. Gusto 전략 분석 & 한국 적용

1. **단일 문제 해결에 집중**: 초기에 급여(Payroll)만 완벽하게 해결

## 10. 기술적 고려사항 (MVP 단계)

> 기술 스택은 별도 결정. 여기서는 아키텍처 원칙만 정의.

### 아키텍처 원칙

| 원칙 | 설명 |
|------|------|
| 세율/요율 DB 관리 | 모든 세금, 보험 요율은 DB 테이블에서 관리. 코드에 하드코딩 절대 금지 |
| 단일 급여 데이터 소스 | 급여 데이터는 한 곳(EmployeeSalaryItem 패턴)에서만 관리 |
| 다중 테넌시 | 모든 쿼리에 companyId 필터 필수 |
| 근태 → 급여 스냅샷 | 근태 데이터는 급여용 스냅샷으로 복사 (원본 변경 영향 차단) |
| PII 암호화 | 주민등록번호 등 민감정보는 AES-256 암호화 |
| Soft Delete | 데이터 삭제 시 deletedAt 플래그 사용 |
| DB 레벨 테넌트 격리 | PostgreSQL RLS + ORM middleware로 companyId 자동 필터링 (application + DB 이중 방어) |
| PII 필드별 암호화 | 주민등록번호 + 계좌번호 AES-256-GCM, canViewSensitive 권한 플래그로 접근 제어 |
| 급여 배치 비동기 | 월급여 일괄 계산은 메시지 큐 기반 비동기 처리 (동기 API 불가) |
| 감사 로그 전수 | 급여/PII/인사/문서/설정 변경 전체 AuditLog 기록 (법적 의무 포함) |
| 계산 스냅샷 불변성 | 급여 계산 시점의 요율/세율 스냅샷 저장, 과거 급여는 당시 법 적용 |
| 파일 스토리지 보안 | 민감 문서 암호화 저장, presigned URL 단기 만료, UUID 파일명 |

### s0에서 배운 기술적 교훈

| 교훈 | s0 문제 | s1 해결책 |
|------|---------|----------|
| 프론트 하드코딩 금지 | 프론트에서 세율 직접 계산 → 연도 변경 시 깨짐 | 모든 계산은 백엔드 API 호출 |
| 급여 데이터 단일화 | User.baseSalary + EmployeeSalaryItem 이중 관리 → 불일치 | 별도 급여 테이블만 사용 |
| 외출 공제 누락 | 외출 승인은 되지만 급여에 미반영 | 근태 스냅샷 생성 시 외출 데이터 포함 |
| 연차 계산 복잡도 | 입사일 기준 / 회계연도 기준 동시 지원 → 로직 비대 | MVP에서는 입사일 기준만 지원 (Phase 2에서 확장) |
| 수당/공제 계산식 | mathjs 평가 + 한글 변수명 → 디버깅 어려움 | 계산식 엔진 단순화, 테스트 커버리지 90%+ |
| 수식엔진 과설계 | mathjs + 한글변수명 → 파싱 에러, 디버깅 불가 | 핵심 법정 계산은 TypeScript 함수, 사용자 정의만 expr-eval |
| 통상임금 미산정 | 기본시급(기본급/209)만 사용, 통상임금 개념 없음 | 통상임금 산정 단계 신설, isOrdinaryWage 플래그 |
| 보험 연중 변경 미대응 | 연도 단위 요율만 관리, 7월 변경 대응 불가 | effectiveStartDate/EndDate 기반 기간별 요율 관리 |
